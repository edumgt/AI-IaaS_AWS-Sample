# Chapter 10. 최종 프로젝트와 확장 로드맵

## 챕터 개요
요구사항 정의, 아키텍처 문서화, 배포/운영 체계

## 학습 목표
- 학습 내용을 통합해 미니 프로젝트 완성
- 학습 결과를 문서화하고 재현 가능한 실습 절차를 확보

## 핵심 이론
- 입력→저장→분석→리포트 흐름 설계
테스트 전략
발표 자료 구성

## 실습 커리큘럼
- 1) 아키텍처 다이어그램 작성
2) 엔드투엔드 데모
3) 개선 백로그 정리

## 산출물
- 실행 명령어와 결과 로그
- 실패 원인/개선안을 포함한 회고 노트

## 완료 기준
- 이론 질문 5개 이상에 답변 가능
- 실습 3개를 독립적으로 재현 가능
- 다음 챕터 선수 조건 충족
